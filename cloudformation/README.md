# CloudFormation — Student Records infrastructure

Native AWS CloudFormation equivalent of the Terraform stack. Same architecture:
VPC (3-tier subnets across 2 AZs), ALB, EC2 Auto Scaling Group, RDS MySQL,
Secrets Manager, IAM, and CloudWatch alarms — in a single template.

## Deploy

```bash
aws cloudformation deploy \
  --template-file student-records.yaml \
  --stack-name student-records \
  --capabilities CAPABILITY_NAMED_IAM \
  --parameter-overrides \
      AppRepoUrl=https://github.com/shatchakra69/aws-student-records-ha.git
```

Get the application URL once the stack is `CREATE_COMPLETE`:

```bash
aws cloudformation describe-stacks --stack-name student-records \
  --query "Stacks[0].Outputs[?OutputKey=='ApplicationURL'].OutputValue" --output text
```

Instances take ~3–5 minutes to bootstrap and pass health checks.

## Tear down

```bash
aws cloudformation delete-stack --stack-name student-records
```

## Validate locally

```bash
pip install cfn-lint
cfn-lint student-records.yaml
```

## Notes

- The database password is generated by `AWS::SecretsManager::Secret` and wired
  into RDS with a dynamic reference — it never appears in the template or in
  CloudFormation parameters/outputs.
- `DeletionPolicy: Delete` on the database keeps teardown simple for a demo.
  Switch to `Snapshot` (or `Retain`) for production.
